# Code Style

## Для всех языков программирования

1. Все комментарии в коде должны быть на английском языке. Исключение: перевод некоторых понятий
  с английского на русский, например, "teeth = зубья".

## Для Ruby

1. Из всех видов callbacks в моделях ActiveRecord разрешено использовать только эти:
  * after_initialize
  * before_validation
  Их можно использовать только для установки значений по умолчанию в поля (атрибуты) модели.
  Для установки простых значений (числа, nil, true, false) предпочтительно использовать attribute:
  `attribute :active, :boolean, default: true`
2. Из кода моделей ActiveRecord не должно быть обращений к сервисным объектам, mailers, jobs,
  serializers.
3. Запрещено использовать concerns в моделях ActiveRecord. Предпочтительно размещать повторяющийся
  код DSL моделей в файлах моделей, методы валидации - в классах-потомках ActiveModel::Validator.

## Для JavaScript / TypeScript

Если есть код на React / Vue / Svelte:

1. По возможности выносим код на JavaScript / TypeScript из файлов
  `*.jsx / *.tsx / *.vue / *.svelte` в файлы `*.js / *.ts`.
